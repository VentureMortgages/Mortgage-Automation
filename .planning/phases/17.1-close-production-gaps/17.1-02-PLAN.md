<objective>
Create a script/endpoint that reads all CRM contacts and all Drive client folders, fuzzy-matches them by name, and populates a Google Sheet with the best-guess pairings. Does NOT update CRM — just builds the spreadsheet for human review. Taylor will confirm matches manually, then we apply them.
</objective>

## Current Gap

Existing CRM contacts (clients who were in the system before go-live) don't have Drive folder IDs linked. When docs arrive for these clients, the system can't file to the right folder — docs fall back to the root Mortgage Clients folder. This is the biggest blocker for Cat using the system day-to-day.

## Approach

Two-phase process:
1. **This plan:** Build a spreadsheet with best-guess matchings for human review
2. **Later (after Taylor confirms):** Apply confirmed matchings from the spreadsheet to CRM

### Spreadsheet Columns
| CRM Contact Name | CRM Contact ID | CRM Email | Drive Folder Name | Drive Folder ID | Match Confidence | Already Linked? | Notes |
|---|---|---|---|---|---|---|---|

<tasks>
<task type="auto">
  <name>Build backfill spreadsheet generator</name>
  <files>src/admin/backfill-spreadsheet.ts, src/webhook/server.ts</files>
  <action>
  1. Create `src/admin/backfill-spreadsheet.ts`:
     - Fetch all CRM contacts from GoHighLevel (paginated)
     - List all top-level folders in DRIVE_ROOT_FOLDER_ID
     - For each CRM contact, check if they already have a Drive folder ID set
     - For contacts WITHOUT a folder ID, fuzzy-match contact name against Drive folder names
       - Use folder naming convention: "LastName, FirstName" or "Last1/Last2, First1/First2"
       - Score each potential match (exact=1.0, close=0.8, partial=0.5)
     - For contacts WITH a folder ID, mark as "Already Linked" and verify the folder still exists
     - Write all rows to the Google Sheet (clear existing data first)
     - Sort: Already Linked first, then by match confidence descending

  2. Register endpoint in server.ts:
     - POST /admin/backfill-spreadsheet
     - Returns count of contacts processed, matches found, already-linked

  IMPORTANT: This endpoint ONLY writes to the spreadsheet. It does NOT update any CRM contacts.
  </action>
  <verify>Hit the endpoint and verify spreadsheet is populated</verify>
  <done>
  - [ ] All CRM contacts appear in spreadsheet
  - [ ] Already-linked contacts marked clearly
  - [ ] Best-guess matches shown with confidence score
  - [ ] Drive folder IDs included for easy verification
  - [ ] No CRM data was modified
  </done>
</task>
</tasks>

<success_criteria>
1. Spreadsheet is populated with all CRM contacts and their best-guess Drive folder matches
2. Contacts already linked to a folder are marked as such
3. Match confidence gives Taylor a quick way to verify/reject
4. No CRM contacts are modified — spreadsheet is read-only output
</success_criteria>
