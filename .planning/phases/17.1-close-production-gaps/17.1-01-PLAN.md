<objective>
When a Finmo application includes co-borrowers, upsert ALL borrowers as CRM contacts (not just the main borrower). Each co-borrower gets linked to the same Drive folder as the primary borrower. This enables sender-email matching for co-borrower documents.
</objective>

## Current Gap

In `src/webhook/worker.ts` (line ~106-120), only the main borrower is upserted to CRM via `upsertContact()`. Co-borrowers exist in the Finmo application data and their checklists are generated correctly, but they have no CRM contact. This means:

1. If co-borrower Jane emails her T4, sender signal can't match (no CRM contact with her email)
2. System falls back to reading name from document — unreliable
3. Docs may land in Needs Review unnecessarily

## Fix

After the main borrower upsert in `worker.ts`, loop through `finmoApp.borrowers`, skip the main borrower, and call `upsertContact()` for each co-borrower with the same Drive folder ID.
