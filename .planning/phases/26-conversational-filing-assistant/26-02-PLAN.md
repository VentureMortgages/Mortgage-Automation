1. Create `src/intake/reply-parser.ts`:

   **Types:**
   ```typescript
   export type ReplyAction = 'select' | 'create_new' | 'skip' | 'unclear';

   export interface ReplyParseResult {
     action: ReplyAction;
     selectedIndex: number | null;   // 0-based index into the options array
     selectedOption: string | null;   // The folder name selected (for logging)
     confidence: number;              // 0.0-1.0
     error?: string;                  // If parsing failed
   }
   ```

   **extractReplyText(fullBody: string): string**
   - Split body into lines
   - Iterate lines and stop at first occurrence of:
     - Line matching `/^On .+ wrote:$/` (Gmail quote marker)
     - Line starting with `>` (quoted text)
     - Line that is exactly `--` (signature delimiter)
   - Return all lines before the marker, joined and trimmed
   - If no marker found, return full text trimmed

   **Gemini schema (ResponseSchema):**
   ```typescript
   const replyParseSchema: ResponseSchema = {
     type: SchemaType.OBJECT,
     properties: {
       selectedOption: {
         type: SchemaType.STRING,
         description: 'The folder name selected, or null',
         nullable: true,
       },
       selectedIndex: {
         type: SchemaType.NUMBER,
         description: 'Zero-based index of selected option from the list',
         nullable: true,
       },
       action: {
         type: SchemaType.STRING,
         enum: ['select', 'create_new', 'skip', 'unclear'],
         description: 'What the user wants to do',
       },
       confidence: {
         type: SchemaType.NUMBER,
         description: 'Confidence in interpretation (0.0 to 1.0)',
       },
     },
     required: ['action', 'confidence'],
   };
   ```

   **parseFilingReply(replyText: string, options: Array<{ folderName: string }>): Promise<ReplyParseResult>**
   - Use same Gemini lazy singleton pattern as body-extractor.ts (import getGenAI or create local one using classificationConfig.geminiApiKey)
   - Build prompt (per RESEARCH.md Example 4):
     ```
     You are parsing a reply from a mortgage broker's assistant who was asked to choose a filing folder for a document.

     The options presented were:
     ${options.map((o, i) => `${i + 1}. ${o.folderName}`).join('\n')}

     The assistant replied: "${replyText}"

     Determine what the assistant wants:
     - If they chose one of the listed options (by number, name, or description), set action to "select" and selectedIndex to the 0-based index
     - If they want a new folder created, set action to "create_new"
     - If they want to skip/leave in Needs Review, set action to "skip"
     - If the reply is unclear or ambiguous, set action to "unclear"

     Set confidence between 0.0 and 1.0. If you are uncertain, set it below 0.7.
     ```
   - Call model.generateContent, parse JSON response
   - Validate: if action is 'select', selectedIndex must be within bounds (0 to options.length-1). If out of bounds, override to action='unclear'
   - Wrap entire function in try/catch. On error, return `{ action: 'unclear', selectedIndex: null, selectedOption: null, confidence: 0, error: err.message }`
