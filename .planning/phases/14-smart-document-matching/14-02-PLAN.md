    ```typescript
    export async function findContactByPhone(phone: string): Promise<string | null> {
      // Normalize: strip non-digits, take last 10 digits
      const normalized = phone.replace(/\D/g, '').slice(-10);
      if (normalized.length < 10) return null;
      const body = { locationId: crmConfig.locationId, query: phone, pageLimit: 5 };
      const response = await crmFetch('/contacts/search', { method: 'POST', body: JSON.stringify(body) });
      const data = await response.json();
      // Exact-match filter: compare last 10 digits of each contact's phone
      const matches = data.contacts.filter(c => c.phone?.replace(/\D/g, '').slice(-10) === normalized);
      if (matches.length !== 1) return null; // ambiguity guard
      return matches[0].id;
    }
    ```

    **Create `src/matching/agent.ts`:**

    Main entry point: `matchDocument(input: MatchInput): Promise<MatchDecision>`

    ```typescript
    export interface MatchInput {
      intakeDocumentId: string;
      classificationResult: ClassificationResult;  // from Gemini classifier (has borrowerFirstName/LastName)
      senderEmail: string | null;
      threadId?: string;
      ccAddresses?: string[];
      emailSubject?: string;
      applicationId: string | null;
      originalFilename: string;
    }
    ```

    **Algorithm (per CONTEXT.md: "Agentic with tools — Gemini gets CRM and Drive API tools and reasons in a loop"):**

    1. **Pre-collect deterministic signals** (before agent loop):
       - Thread signal via collectThreadSignal(threadId)
       - Sender signal via collectSenderSignal(senderEmail)
       - Email metadata signals via collectEmailMetadataSignals(ccAddresses, emailSubject)

    2. **If thread signal found** (Tier 1 deterministic):
       - Per CONTEXT.md: "Thread match is the strongest signal but still passes through the agent"
       - Set thread match as primary candidate, but still run agent to confirm (agent may find conflicting info)

    3. **Build agent system prompt** with:
       - The collected signals so far
       - Classification result (doc type, borrower name, institution, etc.)
       - Signal priority tiers from ROADMAP (Thread > Sender email > Doc content > Filename > CC > Subject > Context)
       - Instruction: "Assign a confidence score 0.0-1.0. If signals conflict, set outcome to 'conflict'."
       - Instruction: "If sender email doesn't match a CRM contact, try search_contact_by_phone if a phone number is available."
       - Instruction: "If sender matches a co-borrower on an application (via lookup_co_borrowers), route to the PRIMARY borrower's folder."
       - Available tools: search_contact_by_email, search_contact_by_name, search_contact_by_phone, get_contact_details, search_opportunities, lookup_co_borrowers

    4. **Agentic loop** (max `matchingConfig.maxAgentIterations` iterations):
       - Send prompt + tools to Gemini
       - If Gemini returns function calls, execute them via executeToolCall and feed results back
       - If Gemini returns a text response (final answer), parse as MatchDecision
       - Break on max iterations (return needs_review)

    5. **Conflict detection**:
       - After agent decides, check if any collected signals point to different contacts
       - If sender matches Contact A but agent chose Contact B based on doc content, set outcome='conflict'
       - Per CONTEXT.md: "Conflicting signals always escalate"

    6. **Determine outcome**:
       - confidence >= autoFileThreshold AND no conflict → 'auto_filed'
       - confidence < autoFileThreshold OR conflict → 'needs_review'
       - No candidates found → 'auto_created'

    7. **Log decision** via logMatchDecision (MATCH-06)

    8. **Return MatchDecision**

    **Gemini response schema** for the final answer (structured output):
    ```typescript
    {
      chosenContactId: string | null,
      chosenOpportunityId: string | null,
      confidence: number,
      reasoning: string,
    }
    ```

    **Fallback for disabled matching:**
    If `matchingConfig.enabled === false`, skip the agent entirely and use the legacy `resolveContactId` behavior (email lookup → name lookup → null). Return MatchDecision with outcome based on whether a contact was found.

    **IMPORTANT:** The agent system prompt should reference signal priority tiers from ROADMAP:
    - Tier 1: Thread match, sender email + single opportunity
    - Tier 2: Doc content name matches one contact, sender email + agent picks opportunity
    - Tier 3: Display name fuzzy, CC/To, subject/body name, doc address, employer
    - Tier 4: Pipeline stage, recency, doc type gaps, professional associations
