Create a test script `scripts/explore-finmo-api.ts` that systematically probes the Finmo API for sync-related endpoints. The script should:

2. Use a known application ID for testing (use Roger Brampton's app: `2f0b2c5f` or fetch the first available from the API)
3. Probe these endpoint patterns (GET requests first, then OPTIONS to discover methods):
   - `GET /api/v1/integrations` or `/api/v1/external-systems`
   - `GET /api/v1/applications/{id}/sync` or `/api/v1/applications/{id}/external-sync`
   - `GET /api/v1/webhooks` or `/api/v1/resthooks` (to see registered hooks and their configs)
   - `GET /api/v1/settings` or `/api/v1/team/settings`
   - `POST /api/v1/applications/{id}/trigger-sync` (with empty body)
   - Check API root `/api/v1/` for an endpoint listing or API discovery document
   - Try `/api/v2/` base as well in case newer endpoints exist
4. For each probe, log: endpoint URL, HTTP method, status code, response body (truncated to 500 chars)
5. Handle 404/403/401 gracefully (these are expected for most probes)

Run with: `npx tsx scripts/explore-finmo-api.ts`

IMPORTANT: Do not use any real client PII. Only use applicationId for API exploration. Log only endpoint metadata, not response content that might contain PII.

The script is a research tool, not production code. Keep it simple.
