<objective>
Add fuzzy Drive folder matching before auto-creating new folders, preventing duplicate folder creation for clients with hyphenated names, compound names, or different name orderings.

Purpose: The Wong-Ranasinghe incident created 3 duplicate folders because `findOrCreateFolder()` uses exact name matching. A search for "RANASINGHE, SRIMAL" didn't find the existing "Wong-Ranasinghe, Carolyn/Srimal" folder. This plan adds fuzzy matching that normalizes names and checks word overlap before creating anything new.

Output: New `folder-search.ts` module, modified `auto-create.ts`, comprehensive tests.
</objective>

1. **Create `src/matching/folder-search.ts`** with these exports:

   **`normalizeName(name: string): string[]`**
   - Lowercase the input
   - Replace all punctuation (hyphens, commas, slashes, periods) with spaces
   - Split on whitespace
   - Filter out empty strings and common filler words
   - Return array of name tokens (e.g., ["wong", "ranasinghe", "carolyn", "srimal"])

   **`fuzzyNameMatch(searchName: string, folderName: string): boolean`**
   - Normalize both names into token arrays
   - Require ALL tokens from the search name to appear in the folder name tokens (exact word match, not substring)
   - This handles: "RANASINGHE, SRIMAL" -> tokens ["ranasinghe", "srimal"] both found in ["wong", "ranasinghe", "carolyn", "srimal"]
   - Return true only if ALL search tokens match (prevents "Smith" matching "Smithson")

   **`searchExistingFolders(drive: DriveClient, clientName: string, rootFolderId: string): Promise<{ folderId: string; folderName: string } | null>`**
   - Extract the last name from `clientName` (first token of normalized name, since format is "LastName, FirstName")
   - Use Drive API `files.list` with `name contains '{lastName}'` query in the root folder (case-insensitive, API-side filtering)
   - Get all matching folders (up to 100)
   - For each result, run `fuzzyNameMatch(clientName, result.name)` for precise filtering
   - If exactly 1 match: return `{ folderId, folderName }`
   - If 0 matches: return null
   - If 2+ matches: log warning with all matches, return null (route to Needs Review downstream)
   - Import `escapeDriveQuery` from `src/classification/filer.ts`
