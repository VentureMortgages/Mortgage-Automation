4. **Create `src/email/filing-confirmation.ts`** with these exports:

   **Types:**
   ```typescript
   export interface FilingResult {
     intakeDocumentId: string;
     originalFilename: string;
     borrowerName: string | null;
     docTypeLabel: string;
     filed: boolean;
     folderPath: string | null;  // Human-readable folder path (e.g., "Wong-Ranasinghe/Income/")
     manualReview: boolean;
     reason: string | null;      // Reason for needs review
   }

   interface MessageContext {
     gmailMessageId: string;
     gmailThreadId: string;
     gmailMessageRfc822Id: string | null;
     senderEmail: string;
     emailSubject: string;
     totalExpected: number;
   }
   ```

   **`recordFilingResult(result: FilingResult, context: MessageContext): Promise<void>`**
   - Store the result in a Redis hash keyed by `filing-results:{gmailMessageId}`
   - Hash field: `intakeDocumentId`, value: JSON.stringify(result)
   - Also store the message context in `filing-context:{gmailMessageId}` (set once, with NX flag)
   - Set TTL on both keys: 1 hour (cleanup for incomplete batches)
   - After storing, check if total stored results == totalExpected → if so, call `maybeSendConfirmation`

   **`maybeSendConfirmation(gmailMessageId: string): Promise<void>`**
   - Read all results from `filing-results:{gmailMessageId}` hash
   - Read context from `filing-context:{gmailMessageId}`
   - If results count < totalExpected: return (not all done yet)
   - Build confirmation body using `buildConfirmationBody(results)`
   - Build MIME message:
     - `from`: intakeConfig.docsInbox (docs@)
     - `to`: context.senderEmail
     - `subject`: `Re: ${context.emailSubject}`
     - `body`: confirmation text
     - `contentType`: 'text/plain'
     - `inReplyTo`: context.gmailMessageRfc822Id (if available)
     - `references`: context.gmailMessageRfc822Id (if available)
   - Encode with `encodeMimeMessage()`
   - Send via Gmail API: `getGmailComposeClient(intakeConfig.docsInbox)` -> `gmail.users.messages.send({ userId: 'me', requestBody: { raw, threadId: context.gmailThreadId } })`
   - Clean up Redis keys after successful send
   - Wrap entire function in try/catch — confirmation failure is NON-FATAL (log error, don't crash pipeline)

   **`buildConfirmationBody(results: FilingResult[]): string`** (exported for testing)
   - Plain text format (not HTML — this is machine feedback)
   - Lists each doc with its outcome:
     - Filed: `  OK  {borrowerName} - {docTypeLabel} -> {folderPath}`
     - Needs Review: `  !!  {borrowerName} - {docTypeLabel} -> Needs Review ({reason})`
     - Error: `  XX  {originalFilename} -> Error`
   - Footer: `-- Venture Mortgages Doc System`
   - Use simple ASCII indicators (OK, !!, XX) instead of emoji for reliability

     // Phase 25: Record filing result for confirmation email
     if (job.data.source === 'gmail' && job.data.gmailMessageId && job.data.totalAttachmentCount) {
